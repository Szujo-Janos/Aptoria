# Aptoria Private License Issuer Tool

Current version: **v0.0.57**

## Purpose

`v0.0.55` added runtime license enforcement and `v0.0.56` added license request/admin management. `v0.0.57` completes the first operational loop by adding a **private-side issuer toolkit**.

The runtime can now:

1. generate a license request;
2. accept a public key;
3. upload and verify a signed `aptoria-license.json` file;
4. block invalid, expired or mismatched portable runtimes.

The issuer toolkit can now:

1. generate an RSA keypair;
2. read `license-request.json` from a customer/runtime;
3. create a license payload;
4. bind it to machine / USB / machine-or-USB fingerprints;
5. sign it with an RSA private key;
6. verify the output before sending it back.

The private key must never be shipped inside Aptoria, committed to GitHub or sent to customers.

## Recommended issuer workflow

### 1. Generate issuer keypair

Run this on the issuer's private/admin machine:

```powershell
php tools\license-issuer\generate-keypair.php --out=tools\license-issuer\keys --name=aptoria-license
```

Output:

```text
tools/license-issuer/keys/aptoria-license-private.pem
tools/license-issuer/keys/aptoria-license-public.pem
```

Keep the private key offline/private.

Install or send only the public key to the runtime:

```text
storage/app/license-public.pem
```

or paste it in:

```text
Program Settings → License Management → Public key
```

---

### 2. Customer generates license request

From the Aptoria runtime:

```powershell
C:\xampp\php\php.exe artisan aptoria:license-request --output=license-request.json
```

Portable helper:

```powershell
.\get-license-request.bat
```

The generated request contains hashed machine and portable-drive fingerprints only. It does not contain raw hardware identifiers.

---

### 3. Issue signed license

Example portable license:

```powershell
php tools\license-issuer\issue-license.php `
  --request=license-request.json `
  --private-key=tools\license-issuer\keys\aptoria-license-private.pem `
  --output=aptoria-license.json `
  --subject="Customer Demo" `
  --issued-to="customer@example.com" `
  --edition=portable `
  --expires=2027-06-20 `
  --binding=machine_or_usb `
  --features="portable_usb,evidence_repository,import_adapter,native_test_evidence,release_gate,client_portal"
```

The output file is:

```text
aptoria-license.json
```

---

### 4. Verify before delivery

```powershell
php tools\license-issuer\verify-license.php `
  --license=aptoria-license.json `
  --public-key=tools\license-issuer\keys\aptoria-license-public.pem `
  --request=license-request.json
```

Expected result:

```text
License verification OK.
```

Only then send the public key and signed license to the customer/runtime.

## Binding modes

| Mode | Use case | Copy resistance |
|---|---|---:|
| `machine` | Fixed self-hosted server/workstation | Stronger |
| `usb` | Strict portable drive license | Stronger for USB, fragile if USB is reformatted |
| `machine_or_usb` | Recommended portable default | Balanced |
| `none` | Trial/internal/demo only | Weak |

For normal portable use, use:

```text
machine_or_usb
```

This lets the runtime pass if either the machine fingerprint or the USB/portable fingerprint matches.

## License payload generated by the issuer

The generated `aptoria-license.json` uses the runtime-supported structure:

```json
{
  "payload": {
    "license_id": "APT-20260620-ABC123",
    "product": "aptoria",
    "edition": "portable",
    "subject": "Customer Demo",
    "issued_to": "customer@example.com",
    "issued_at": "2026-06-20T12:00:00Z",
    "expires_at": "2027-06-20T23:59:59Z",
    "features": [
      "portable_usb",
      "evidence_repository",
      "import_adapter",
      "native_test_evidence",
      "release_gate",
      "client_portal"
    ],
    "fingerprint_binding": {
      "mode": "machine_or_usb",
      "fingerprints": [
        "sha256:...",
        "sha256:..."
      ]
    },
    "issuer": "Aptoria License Issuer",
    "source_request": {
      "request_id": "lrq_...",
      "request_format": "aptoria-license-request-v1",
      "version": "0.0.57",
      "generated_at": "2026-06-20T12:00:00+00:00"
    }
  },
  "signature": "BASE64_RSA_SHA256_SIGNATURE"
}
```

The signature is calculated from the canonical sorted payload, matching `LicenseGuardService::canonicalPayload()`.

## Security boundary

This protects against simple copy-and-run redistribution. It is not unbreakable DRM.

Important limits:

- PHP source can theoretically be modified by someone with code access.
- The private key is the real trust anchor.
- Do not ship the private key.
- Do not let customer packages include issuer `keys/` or signed test licenses.
- For commercial builds, preinstall/pin the issuer public key and avoid giving customers unnecessary public-key replacement rights.

## QA checklist

- Generate a keypair.
- Generate a license from `examples/license-request.example.json`.
- Verify the license with the public key.
- Change one character in the license payload and verify that validation fails.
- Change binding mode to `machine` and verify it only accepts the machine fingerprint.
- Change binding mode to `usb` and verify it only accepts the USB fingerprint.
- Create an expired license and verify the runtime blocks it.
- Confirm no `*.pem`, `keys/`, generated `aptoria-license.json` or private output files are included in release ZIPs.
