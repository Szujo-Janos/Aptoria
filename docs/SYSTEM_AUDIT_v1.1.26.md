# Aptoria v1.1.26 System Audit

Release: **v1.1.26 - Report Versioning & Approval Pass**

## Scope

This release turns generated reports into versioned and approvable decision objects. A report can now be saved with a checksum, source evidence snapshot, lifecycle status and approval metadata before it is handed to a customer or used for release sign-off.

## Added components

- `report_versions` database table
- `App\Models\ReportVersion`
- `App\Services\Reports\ReportVersioningService`
- `App\Http\Controllers\ReportVersionController`
- `resources/views/report_versions/index.blade.php`
- `resources/views/report_versions/show.blade.php`
- `tests/Feature/ReportVersioningApprovalTest.php`

## Main behaviours

- Creates draft report versions from Executive, Technical, Release Readiness or Full Project report sources.
- Stores markdown content and a SHA-256 checksum.
- Captures source scan IDs, snapshot IDs, compare IDs, finding state, release gate IDs, release decision IDs and evidence attachment IDs.
- Supports Reviewed, Approved and Archived lifecycle states.
- Records generated by, approved by, generated at, reviewed at, approved at and archived at metadata.
- Exports saved versions as Markdown, HTML, PDF and JSON.
- Writes audit log events for version creation, approval and archive actions.
