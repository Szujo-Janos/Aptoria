# Aptoria v1.1.32 – Members & Roles Dashboard Redesign

This hotfix redesigns the Project → Members & Roles screen so the access workflow is clear and visually balanced.

## Structure

1. Project access dashboard summary
2. Access workflow explanation
3. Existing internal users that can be added to the project
4. Create new user and add to project
5. Current project members
6. Role permission reference
7. Current user's effective access

## UX rules

- Existing users and new user creation are separate workflows.
- The current project members table is full width and visually distinct.
- Permission codes are never shown directly to users.
- All visible labels must be available in both English and Hungarian.
- Page-load fade/appearance animations remain disabled.
